#!/usr/bin/env -S bunx --bun -p https://github.com/subtleGradient/pad/archive/refs/heads/main.tar.gz pad Appify UI 2026 Canvas Proposal Appify UI 2026 Canvas Proposal Summary Create a fresh SwiftPM macOS document launcher for JSON Canvas files by cloning the proven AppifyUI2026 launcher shape into /Users/tom/Developer/pad/macos/AppifyUI2026-canvas/. The app is named Canvas, owns plain .canvas files, and opens them through the existing PAD Bun runner in a native WKWebView. Document Type Canvas.app uses bundle identifier com.subtlegradient.Canvas and declares .canvas as com.subtlegradient.pad.canvas. The type conforms to public.json. It is a normal file document, not a package, so the app bundle must not set LSTypeIsPackage. Runner Contract The native app never invokes a shell, never reads a manifest, and never executes code from the .canvas file. It resolves system Bun the same way AppifyUI2026 does, starts a Process with explicit argv, and sets PAD_NO_OPEN=1 so AppKit owns the window and WebView. When Canvas.app lives inside a PAD git checkout containing package.json and pad.shebang.tsx in the expected place, it runs the local source as bun /absolute/path/to/pad.shebang.tsx /absolute/document.canvas. This keeps development builds pointed at the working tree. When Canvas.app is copied or installed outside that source checkout, it falls back to a pinned remote PAD package: bun x --bun --package https://github.com/subtleGradient/pad/archive/03823418e3d8059a640b125717e738ed6e81e705.tar.gz pad /absolute/document.canvas. Open URL Contract PAD already prints a compatible stdout line such as file.canvas: http://127.0.0.1:54119/?t=token. Canvas.app should extract the first URL from runner stdout, accept only loopback HTTP(S), and load it in a WKWebView. The app should show native error states for missing Bun, invalid document extension, runner timeout, unsafe URL, and early process exit. Native App Behavior Use AppKit plus WKWebView with one window/controller per opened .canvas file. Finder and open -a launches flow through application(_:open:). Direct app launch shows an open panel restricted to .canvas files. Runner stdout and stderr stream to ~/Library/Logs/Canvas/ and the child process terminates when its document window closes. Deferred Scope V1 only declares the document type and launches the editor. It does not set Canvas.app as the default .canvas handler, does not include Quick Look preview or thumbnail extensions, does not bundle Bun, and does not implement a separate installer command.
